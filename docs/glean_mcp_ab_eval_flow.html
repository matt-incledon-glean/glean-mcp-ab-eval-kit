Glean MCP A/B Eval Kit · Process Flow

How the evaluation runs, end to end

A defensible A/B evaluation of Glean MCP vs. vendor-direct MCP connectors in Claude Code. Same model, same prompts, same machine — usage harvested from real transcripts, graded blind, and aggregated centrally.

Grade

Pass to blind judge

Recorded answers are scored without revealing which arm produced them.

Evaluate · Efficiency

Cost & usage

Input/output tokens, cache tokens, total tokens, list-price cost, tool-call counts.

Evaluate · Quality

Answer quality

Completeness, groundedness, usefulness, and per-pair winner.

Persist

Write to results file

Per-prompt rows stored under results/<participant>/<arm>/<prompt_id>/, then packaged with checksums into eval_submission.zip.

Aggregate

Pass to admin for aggregation

Analysis owner imports each participant zip, then runs grade → report → package to produce aggregate_summary.md, aggregate_rows.csv, and the submission manifest.

Deliver

Share results

Stakeholder readout of efficiency and quality findings across all participants and arms.
